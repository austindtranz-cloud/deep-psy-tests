# DEEP Tests  
**Навигационный sidebar и карточки тестов — визуальная спецификация**

Документ фиксирует **левый sidebar с ветвлениями**, логику состояний и визуальный стиль карточек **на главной странице как карточек тестов**, а не папок. Спецификация собрана как рабочая опора для дальнейшей сборки в Tilda / T123.

**Главный принцип: главная страница показывает тестовые карточки и маршруты. Sidebar слева отвечает за навигацию по категориям, подкатегориям и активному маршруту.**

# 1\. Базовый визуальный контур

Фон всего раздела тестов — #111111. Общий характер интерфейса: dark-premium, спокойный, структурный, без лишней декоративности. Акцентный цвет — #E8D6B3. Вторичный успех/подтверждение — мягкий зелёный диапазон около #89A97A. Белый и почти-белый используются для типографики и тонких разделителей.

## 1.1. Основные дизайн-токены

|     |     |     |     |
| --- | --- | --- | --- |
| Токен | Значение | Где применяется | Комментарий |
| Фон раздела | #111111 | Весь раздел тестов | Глубокий тёмный фон без градиентного шума |
| Основной текст | #F4EEE3 / белый | Заголовки, основной текст на тёмном фоне | Контраст держать высоким |
| Акцент | #E8D6B3 | Теги, hover-акценты, CTA, активные элементы | Использовать дозированно |
| Граница | rgba(232,214,179,0.14–0.28) | Карточки, sidebar, pills | Границы мягкие, не жёсткие |
| Подложка панели | #0F1113 / #131518 | Sidebar, карточки, вложенные блоки | Немного светлее фона |

# 2\. Sidebar слева: роль, состав и состояния

Sidebar — это не декоративная колонка, а главный навигационный и смысловой слой раздела. Он должен показывать, где пользователь находится в структуре тестов, по какой ветке он движется и какой следующий уровень можно раскрыть.

## 2.1. Как называются элементы sidebar

**Sidebar shell:** Внешняя левая панель с общей рамкой, отступами и прокруткой.

**Section label:** Небольшой надзаголовок в верхней части панели. Например: Навигация, Категории, Маршрут.

**Category row:** Строка верхнеуровневой категории. Кликабельна, может раскрываться.

**Branch toggle:** Индикатор раскрытия ветки: chevron / стрелка / плюс-минус.

**Subcategory row:** Строка подкатегории внутри выбранной категории.

**Current route chip:** Плашка текущего маршрута или выбранного режима.

**Support block:** Нижний блок с CTA, короткой помощью или переходом к библиотеке.

## 2.2. Визуальные состояния sidebar

|     |     |     |     |
| --- | --- | --- | --- |
| Состояние | Как выглядит | Цвет/тон | Поведение |
| Default | Спокойная строка без сильной подложки | Текст — светлый, фон почти прозрачный | Ничего не мигает, не шумит |
| Hover | Строка чуть подсвечивается, фон мягко теплеет | Лёгкая акцентная дымка | Показывает кликабельность |
| Active | Текущая категория / подкатегория явно выделена | Акцентная подложка и/или бордер | Одновременно может быть только один active-route |
| Expanded | Ветка раскрыта, дочерние элементы видимы | Chevron повёрнут вниз | Раскрытие плавное, без скачков |
| Completed/Done | Опциональное состояние для пройденного блока | Мягкий зелёный индикатор | Не забивает основной акцент |

## 2.3. Логика ветвления

- Верхний уровень sidebar показывает все верхнеуровневые категории.
- При выборе категории раскрываются её подкатегории, а остальные ветки остаются свернутыми или уходят в secondary state.
- При выборе подкатегории sidebar должен явно показывать путь: категория → подкатегория.
- Если пользователь вошёл в тест по прямой ссылке, sidebar всё равно должен подсветить, к какой категории и подкатегории он относится.
- На мобильном sidebar лучше переводить в off-canvas / drawer-паттерн, чтобы он не съедал рабочую площадь.

# 3\. Главная страница: карточки тестов, а не папки

**Критический принцип: главная не должна визуально выглядеть как файловый менеджер. Это не папки и не директории. Это карточки тестов / маршрутов / исследовательских направлений.**

Карточка на главной должна восприниматься как самостоятельная диагностическая единица: у неё есть название, краткий смысл, примерный объём, время прохождения и понятный следующий шаг. Визуально — это не иконка папки, а premium-карточка сервиса.

## 3.1. Анатомия карточки теста

|     |     |     |     |
| --- | --- | --- | --- |
| Элемент карточки | Что в нём | Как выглядит | Обязательность |
| Kicker / Tag | Короткий ярлык: Личность / Скрининг / Карьера | Небольшая pill-плашка | Да  |
| Title | Название теста | Крупный заголовок 20–28 px в вебе | Да  |
| Meta | Вопросы / время / число шкал | Небольшие pills или строка-метаданные | Да  |
| Description | Практическое описание, что измеряет тест | 2–4 строки без воды | Да  |
| CTA | Пройти тест / Открыть маршрут / Подробнее | Акцентная кнопка или link-button | Да  |
| Route hint | Если это широкий вход: что будет дальше | Вторичная подпись | Опционально |

## 3.2. Визуальный стиль карточек

- Форма: прямоугольник с крупным радиусом (примерно 20–24 px).
- Фон: тёмная панель #0F1113 / #131518, чуть светлее общего фона.
- Граница: тонкая, полупрозрачная, в акцентном диапазоне.
- Тень: мягкая, глубокая, без дешёвого glow-эффекта.
- На hover карточка слегка поднимается и теплеет по границе/подложке.
- Внутри должно быть много воздуха: карточка не может выглядеть забитой.

## 3.3. Типографика карточек

- Основной UI-шрифт — Manrope или близкий современный grotesk.
- Название карточки должно быть выше по иерархии, чем всё остальное.
- Описание карточки не должно превращаться в длинную простыню: лучше компактно и practically-oriented.
- Метаинформация вторичная и должна читаться быстро: 14 вопросов, 8 минут, 2 шкалы.

# 4\. Как соотносятся sidebar и карточки

Sidebar отвечает за навигацию по структуре. Карточки отвечают за выбор конкретного направления, теста или маршрута. Они не дублируют друг друга буквально, а работают в паре.

|     |     |     |
| --- | --- | --- |
| Компонент | Основная задача | Чего не делать |
| Sidebar | Ориентировать пользователя в структуре | Не превращать в второй каталог карточек |
| Карточки на главной | Давать понятные входы в тесты и маршруты | Не стилизовать как папки/директории |
| Breadcrumb / route hint | Показывать, где человек находится | Не делать слишком тяжёлым визуально |

# 5\. Что должно быть на главной визуально

- Верхний hero или вводный экран — отдельно, не смешивать с сеткой карточек.
- Ниже — сетка карточек тестов / маршрутов по 2–3 в ряд на desktop, 1 в ряд на mobile.
- Карточки могут быть сгруппированы визуально по категориям, но каждая остаётся самостоятельной тестовой карточкой.
- Отдельные category headers допустимы, но они не заменяют карточки.
- Если нужен “широкий вход”, карточка должна явно писать: широкий скрининг, базовый профиль, углубление и т.д.

# 6\. Адаптивное поведение

- На desktop sidebar закреплён слева и не мешает чтению центральной части.
- На tablet sidebar может становиться компактнее по ширине, но логика ветвления сохраняется.
- На mobile sidebar превращается в drawer / off-canvas-панель по кнопке.
- Карточки на mobile занимают всю ширину контейнера и получают уменьшенные, но не зажатые отступы.
- Текст не должен вылетать за границы карточек, горизонтальный скролл недопустим.

# 8\. Что делать и чего не делать

|     |     |
| --- | --- |
| Делать | Не делать |
| Показывать структуру через sidebar и состояние active/expanded. | Не дублировать ту же структуру бессмысленно в трёх местах сразу. |
| Использовать карточки тестов как сервисные entry-points. | Не стилизовать карточки как файловые папки. |
| Держать воздух и тишину в интерфейсе. | Не забивать экран иконками, тенями и градиентами. |
| Ясно показывать маршрут пользователя. | Не заставлять человека гадать, где он находится. |
| Делать mobile-first реальным, а не декоративным. | Не оставлять desktop-логику без адаптации на mobile. |

# 9\. Итоговая формула интерфейса

Идеальная логика такая: слева — спокойный, структурный sidebar с ветвлениями; в центре — сетка тестовых карточек и контент; внутри карточек — ясный practically-oriented вход; внутри маршрута — последовательное углубление. Главная показывает тесты как диагностические сервисы, а не как папки или файловую иерархию.
