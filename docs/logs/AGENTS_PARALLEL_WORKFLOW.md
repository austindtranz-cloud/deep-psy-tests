# Протокол параллельной работы агентов (Git Worktree)

**ВАЖНО ДЛЯ ВСЕХ АГЕНТОВ:** Мы работаем в рамках одного корневого репозитория (`d:\Рабочий стол\Тесты DPS`). Если несколько агентов одновременно выполняют команду `git checkout <branch>` в данной директории, они перезаписывают `working tree` (рабочее дерево) друг друга в реальном времени. Это приводит к:
1. Потере незакоммиченных файлов (ваших `WORKLOG.md`, `PLAN.md`, `HANDOFF.md`).
2. Внезапным "грязным" коммитам (вы коммитите чужие файлы в свою ветку).
3. Постоянным конфликтам и гонкам состояний (race conditions).

## Как правильно создавать ветку и работать (Worktree Workflow)

Чтобы этого избежать, мы переходим на использование механизма **Git Worktree**. Он позволяет каждому агенту иметь **собственную физическую папку**, привязанную к той же истории `.git` родителя, но со своей независимой файловой системой (`working tree`).

### Шаг 1. Отказ от базового `git checkout`
При старте новой сессии **запрещено** создавать ветку прямо в корне проекта (через `git checkout -b <новая-ветка>`).
Вместо этого мы используем директорию `_worktrees/` (она уже добавлена в `.gitignore`).

### Шаг 2. Создание изолированного Worktree
Используйте команду для создания новой папки под свою ветку:
```powershell
git worktree add ./_worktrees/agent-<ваша_задача>-<дата> -b agent/<ваша_задача>-<дата>
```
*Например: `git worktree add ./_worktrees/agent-ui-fix-2026-03-28 -b agent/ui-fix-2026-03-28`*

### Шаг 3. Переход в свою папку
После создания Worktree, **смените рабочую директорию** для всех последующих операций с файлами и запуска команд (`Cwd` / `cd`):
```powershell
cd ./_worktrees/agent-<ваша_задача>-<дата>
```
*Все инструменты редактирования файлов (`write_to_file`, `replace_file_content` и т.д.) теперь должны ссылаться на пути внутри этой новой папки!*

### Шаг 4. Инициализация файлов отчета
Создайте ваши `WORKLOG.md`, `PLAN.md` и `HANDOFF.md` внутри вашей папки (например, `_worktrees/agent-.../agents/dev/WORKLOG.md`). 
Делайте `git add` и `git commit` только изнутри вашего Worktree. Git автоматически привяжет их к вашей безопасной ветке. Так вы не затронете работу других агентов.

### Шаг 5. Завершение работы
Когда ваша задача завершена и `HANDOFF.md` актуализирован, папка Worktree остается как есть, чтобы вы или другой агент могли в будущем к ней вернуться, или вы можете удалить ее через `git worktree remove`, когда ветка будет смерджена. Основной репозиторий при этом остается в чистоте.

> [!CAUTION]
> **Не забывайте**: Не выполняйте `git push` или изменения в ядре проекта без предварительного обновления/пулла, так как другие агенты могут параллельно вносить изменения в свои `worktree` и пушить в общий удаленный репозиторий.
