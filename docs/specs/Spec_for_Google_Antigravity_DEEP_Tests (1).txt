DEEP PSY TESTS — MASTER SPEC FOR GOOGLE ANTIGRAVITY

This package includes a detailed DOCX spec. Below is the copy-paste master prompt excerpt.

Ты проектируешь и пишешь production-ready код для Tilda (T123) под раздел психологических тестов DEEP.

Контекст:
- Бренд: DEEP PSY SOLUTIONS.
- Стиль: dark premium / scientific premium.
- Основной фон всей системы: #111111.
- Все ключевые UI-блоки должны выглядеть дорого, чисто, понятно и без синего SaaS-оттенка.
- Заказчик вручную соберёт только шапку, первый экран и, возможно, отдельные текстовые hero-блоки. Всё остальное должно быть кодом внутри T123.

Что нужно выдать:
1) код для страницы /tests;
2) код для страницы /tests/personality;
3) shared CSS/JS layer или понятное разделение общего кода;
4) инструкцию по вставке в Tilda.

Ключевое архитектурное решение:
- Не пытайся строить весь сайт внутри одной страницы.
- Предполагай, что /tests и /tests/personality — реальные Tilda-страницы с реальными slug.
- Внутри них ты собираешь кодом весь контентный и функциональный слой после hero.
- Query params можно использовать только для deep-link, фильтрации и стартового состояния, но не как замену реальных SEO-страниц.

Адаптив:
- >1200: horizontal padding 40px
- 960: horizontal padding 40px
- 640: horizontal padding 20px
- <=480: horizontal padding 10px
- mobile-first
- никакого горизонтального скролла

Цвета:
- bg: #111111
- text main: #F4EEE3
- text muted: #A89F91
- accent: #E8D6B3
- accent hover: #D9C29A
- success: #8FAE7B
- border: rgba(232,214,179,0.14)

Главная страница /tests должна содержать:
- 10 верхнеуровневых категорий:
  1. Исследование личности
  2. Психические функции
  3. Адаптация и дезадаптация
  4. Психиатрия
  5. Социальные и межличностные тесты, секс
  6. Индивидуальное карьерное консультирование
  7. Командное консультирование
  8. Организационное консультирование
  9. Психоаналитическая диагностика
  10. Оценка эффективности терапии
- под каждой категорией human-readable question line
- editorial карточки категорий
- блок “Начать с запроса”
- библиотеку тестов по русскоязычным названиям
- библиотеку по ключевым словам
- CTA и юридический дисклеймер

Страница /tests/personality должна содержать:
- категорию “Исследование личности”
- подкатегории:
  - структура личности
  - акцентуации
  - темперамент
  - автономия
  - самооценка
  - личностные черты
  - юнгианские / психоаналитические
- для каждой подкатегории: researched cards тестов
- внутри маршрута показать:
  - широкий вход
  - уточнение
  - углубление
  - глубина
- показать next-step logic и cross-routes в другие разделы

Research rules:
- исследуй по интернету только надёжные источники
- если лицензия / public domain / open-source аналог неясны, не вставляй полный тест целиком
- на карточке честно помечай статус: validated / needs legal review / official only / open analogue recommended
- не придумывай количество вопросов и длительность, если источник не подтверждён
- не копируй защищённые опросники целиком

Важно:
- На странице personality нужен хотя бы один demo fully-working runner и demo result screen, чтобы проверить архитектуру.
- Остальные тесты могут быть пока как researched cards и route points.
- Результат должен быть скриншотопригодным, понятным и красивым.
- Используй графики, бары или radar только если это реально улучшает понимание.
- После результата покажи:
  - что это значит
  - риски при текущем уровне
  - следующий шаг
  - альтернативный шаг
  - конверсионный шаг

Поведение CTA:
- норма / ниже нормы -> контент + следующий тест
- риск -> мягкое приглашение к продукту / консультации
- клиника -> прямой оффер обратиться к администратору

Разложи код на секции:
- design tokens
- data registries
- ui components
- routing helpers
- localStorage
- demo runner
- result renderer
- chart renderer

Финальный ответ верни так:
1. коротко объясни архитектуру;
2. дай код для /tests;
3. дай код для /tests/personality;
4. дай shared CSS/JS или понятный способ вынести общее;
5. дай инструкцию по вставке в T123;
6. перечисли, какие тесты ты реально исследовал и какие оставил как placeholders из-за лицензий / проверки.
