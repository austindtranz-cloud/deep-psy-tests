## 2. Логика загрузки (js.txt)

Система использует **Smart Loader**, который:
1. Проверяет актуальную версию коммита в Github (кэширует на сессию).
2. Подключает CSS и JS файлы в строгом порядке для соблюдения зависимостей.

**Порядок инъекции скриптов:**
1. `style.css`
2. `test_registry.js`
3. `router.js`
4. `quiz_core.js`
5. `ui_controller.js`
6. `integrations.js`
7. `engine.js` (Инициация `DEEP_CORE`)
8. `engine_results.js`
9. `sidebar.js`

## 3. Краткое описание модулей Core

### `router.js`
Отвечает за "красивые" URL и автоматическое переключение тем (категорий) без явной настройки в Tilda. Ищет в пути `/tests/personality` ключевые слова и подгружает данные.

### `quiz_core.js`
Только логика. Не знает про HTML. Хранит ответы, считает баллы, управляет `localStorage` ключом `deep-tests-engine-v4`.

### `ui_controller.js`
Генерирует HTML-карточки тестов, сетку дашборда Matilda Hybrid, экраны вопросов. Содержит:
- **Глобальную делегацию событий** (`_attachGlobalDelegation`) — единый обработчик для кликов, тултипов, переключения вкладок
- **Тултипы-порталы** — `mouseover/mouseout` делегация, рендер в `document.body` (не внутри карточки)
- **Letter Quick-Picker** — попап алфавита при клике на буквенный заголовок
- **Тэги 3-column grid** — аккордеон (один-открыт-другие-закрыты), короткие ключевые слова
- **SVG-иконки** и оверлей блокировки скролла
