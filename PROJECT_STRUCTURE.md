## 1. Что это

**DEEP PSY TESTS** — модульный каталог психологических тестов, встроенный в сайт на Тильде ([deeppsysolutions.com](https://deeppsysolutions.com)). Код хостится на **GitHub Pages**. 
Ключевая особенность v11.8 — полностью автономный **Smart Cache-Busting Loader**, который сам ходит в API GitHub, узнаёт хэш последнего коммита и сбрасывает кэш у всех JS/CSS скриптов, а во время загрузки показывает чёрный прелоадер.

## 3. Как работает загрузка (Smart Loader)

1. Тильда выполняет код из `Алиас/js.txt`.
2. Лоадер мгновенно рисует **тёмный прелоадер (fullscreen overlay)**, закрывая дёргания вёрстки и подгрузку тяжелых скриптов.
3. Лоадер делает быстрый запрос `fetch("https://api.github.com/repos/austindtranz-cloud/deep-psy-tests/commits/main")`.
4. Из ответа берет актуальный `sha` (хэш последнего коммита).
5. Создаёт теги `<script src=".../engine.js?v={sha}">` и подгружает ядро. Таким образом, кэш сбрасывается только тогда, когда реально выходит новый код.
6. После загрузки всех скриптов вызывается `deepRenderCatalog()`, прелоадер плавно исчезает (fade-out 500ms).

## 4. Движок и UX-фичи (v11.8)

- **Плавный авто-переход**: При клике на вариант ответа визуально подсвечивается радио-кнопка (класс `.is-selected`), движок ждёт комфортные `150мс` и только потом плавно переключает следующий вопрос. Это даёт нейромышечное подтверждение выбора.
- **Интеграция клавиатуры**: Нажатия клавиш `1-9` (в т.ч. NumPad) программно симулируют настоящий клик мыши `.click()` по варианту ответа, вызывая ту же плавную анимацию. Конфликты старых слушателей `Enter` убраны. В полях ввода (email, имя) клавиатурные шорткаты игнорируются.
- **Self-Healing Sidebar (Самоочищающееся меню)**: Скрипт `sidebar.js` автоматически находит и удаляет устаревшие HTML-блоки бокового меню (жёстковшитые кнопки и `<aside>`), если пользователь забыл стереть их в Тильде при обновлении. И только потом рендерит свой чистый DOM. Это исключает дублирование кнопок и поломки CSS.
- **Сброс сессии (Пройти заново)**: Если юзер хочет пройти тест повторно, `engine.js` полностью очищает ответы и сбрасывает статус отправки `session.sent = false`. Форма лидогенерации снова становится активной.
- **Лид-форма (Форма отправки)**: Кнопка «Получить отчёт» при отправке блочится с текстом «Отправка...». При успехе она стилизуется в акцентный зелёный цвет `#52b788` жестко через код (с использованием `setProperty(..., 'important')` для обхода системного CSS Тильды) и меняет текст на «Отправлено».

## 5. Формат оцифрованных тестов

Существует 2 типа логики тестов:
1. **Generic (Простые)**: Описание логики зашито прямо в `options[].value / score`. Баллы считаются автоматически через простое суммирование внутри `engine.js` -> `calculateGenericResult()`. Пример: `rosenberg` (10 вопросов).
2. **Custom (Сложные)**: Имеют функцию `calculateResult(answers)`. Движок передает туда map ответов `{"q1": value}`, а функция возвращает сложный массив данных об очках/шкалах/интерпретациях, которые `engine_results.js` затем рендерит. Примеры: `leongard` (коэффициенты), `smil_566` (сырые баллы -> T-баллы -> K-коррекция -> графики).

Остальные 47+ тестов в реестре (у которых нет поля `questions`) считаются не до конца оцифрованными. При клике на такой тест появляется Alert-предупреждение.

## 7. Деплой и обновления (Инструкция для разработчика)

Благодаря внедрению Smart Cache-Busting Loader в v11.8, порядок обновления проекта стал полностью автоматическим и не требует входа в панель Tilda CMS.

1. Локально модифицируем код (JS / CSS / HTML / Добавление новых тестов в папку data).
2. Выполняем коммит:
   ```bash
   git add -A
   git commit -m "update engine"
   git push origin main
   ```
3. Github Pages обновляется в фоне за 30-40 секунд.
4. **НЕ НУЖНО** ничего менять в Тильде! Лоадер при следующей загрузке страницы сам увидит свежий хэш-коммит API Github, сбросит browser cache у клиентов и загрузит самый свежий код.
